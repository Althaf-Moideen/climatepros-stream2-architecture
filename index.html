1. System Overview

ClimatePros is an AI-assisted estimating platform for commercial refrigeration and HVAC bids. Estimators upload engineering drawing packages; the platform extracts a structured takeoff with a two-model AI pass, prices it deterministically, routes it through a review-and-approval workflow, and delivers the FieldJetX Excel (the only file sent to FieldJetX) plus an in-app summary PDF. It runs on Microsoft Azure.

🧮

Design Principle: LLMs do quantities, code does dollars

Models only extract scope & quantities; every price is computed by pure, auditable code.

Gemini extracts, Claude verifies the money fields, a deterministic reconciler merges them, and a deterministic engine prices the result — so every estimate is reproducible and traceable, and no model ever emits a price.

Request Flow

  1. User signs in via Entra ID (OIDC + MFA); the SPA calls the API with a Bearer token.
  2. Traffic passes Front Door + WAF → App Gateway (TLS, filtering, rate-limiting) to the API.
  3. The API validates the token + RBAC, then serves sync actions or enqueues an analyze job on Service Bus.
  4. Workers (autoscaled by KEDA) run Gemini → Claude → reconcile, writing results to PostgreSQL.
  5. On approval, Export pushes the Excel to FieldJetX and renders the in-app summary PDF.

5. Application Workflow (Data Flow)

1

Upload & Normalise — deterministic, no AI

Detect format → convert everything to PDF (DOC via LibreOffice, CAD via exporter, images wrapped) → render each page (200 DPI, auto-bumped to 300 for dense schedule/E-size sheets) → obtain the text by the most accurate route: native text extraction for born-digital (vector) PDFs (exact + free); scans, photos and image-only pages carry no text layer, so they are marked needs_vision and read directly by Gemini vision in Step 2 (more accurate than OCR on messy drawings) → quality gate flags blank/blurry/garbled pages instead of passing them silently. Uniform per-page images (+ native text where present) land in Blob + PostgreSQL so the models see consistent inputs for every document type.

2

Analyze — two-model takeoff (async worker)

Gemini extracts into a constrained JSON schema (validated + auto-retried), guided by a domain prompt + few-shot examples, each item grounded to its source page and read from high-DPI schedule crops. Claude independently re-sweeps for missed items (protects recall) and re-checks the money fields (sizes, quantities, N/E/R/X status), assigning confidence and flagging inconsistencies — never a silent overwrite. A deterministic reconciler merges: agreement boosts confidence, disagreements/misses become RFIs. No prices.

3

Estimate

Deterministic engine maps each item to a rate line (no rate → unpriced flag, never $0), prices by status (N/E/R/X) in Decimal, and pins the rate-card version → honest + draft totals, fully re-runnable. No model call.

4

Review & Approval

Estimator submits; reviewer/manager approves or rejects with sign-off; audit-logged state machine.

5

Deliver

Excel pushed to FieldJetX (the only file sent); in-app summary PDF (findings + bid details) generated for view/download.

7. Core Services

What each service does — plain language, with the real terms kept.

📥

1 · Intake — get the drawings ready

any file → clean pages · no AI

Takes any file — PDF, Word, CAD, scan or photo — and turns it into clean, consistent pages. Word and CAD are first converted to PDF; each page is rendered as an image (200–300 DPI). It then gets the text the most accurate way: native text straight from normal (vector) PDFs, or Gemini vision for scans and photos. Blurry or blank pages are flagged (needs_review) instead of guessed. No AI decisions here — just preparation.

Works for every file type: vector PDFs use their own text, scans and photos go to Gemini vision (most accurate on messy drawings), and Word/CAD are converted first. Unreadable pages are flagged for a person — so a bad scan never turns into a wrong estimate.
🤖

2 · Analyze — two AIs read the pages

Gemini + Claude

Gemini reads the pages and extracts the takeoff — equipment, sizes, quantities and each item's status (N/E/R/X) — as structured data (JSON), with every item linked back to the page it came from. Claude then verifies the price-driving fields and looks for anything Gemini missed, giving each item a confidence score. If they disagree it becomes an RFI (a question for a human), never a silent overwrite. No prices here.

🔀

3 · Reconcile — merge into one list

plain code · no AI

Merges Gemini's and Claude's results into one clean list of bid_items. It first matches the same item across both, then merges field by field: agreement raises confidence, a confident value fills a blank, and any disagreement or missed item becomes an RFI. Fully deterministic — the same input always gives the same result, and every decision is logged.

Built to scale: pages run in parallel on background workers (Celery, autoscaled by KEDA), so 5 or 500 pages follow the same path. Repeat uploads are cached, and the AI models are pinned but swappable — a newer/cheaper one drops in without changing anything else, holding the <5-min target and controlling cost.
💵

4 · Estimate — work out the cost

pure code · no AI

Prices the job with plain code — no AI touches the dollars. Each item is matched to a rate card line; an item with no rate is flagged unpriced, never counted as $0. It produces two totals — an honest total (confirmed items) and a draft total (includes uncertain/RFI items) — using Decimal money so there's no rounding drift. The rate version is pinned to the bid, so a re-run always gives the same figure and every dollar is traceable.

✅

5 · Review & Approval — people sign off

human check

The bid moves through clear states: draft → in_review → approved → finalised. The estimator submits; a reviewer or manager approves — and the approver can't be the estimator. Open RFIs or unpriced items block final sign-off (but not drafting), so nothing ships with a hidden gap. Every step is kept in an audit trail.

📤

6 · Deliver — send the result

Excel + PDF

Pushes an Excel file to FieldJetX — the only file that leaves the system. A summary PDF of the findings and bid details is also created in-app to view or download.

👥

Clients, Dashboard & Analytics

multi-tenant

Client registry with per-client bid history, search, reuse; KPI dashboard and win/margin analytics.

10. Applied-AI Engineering & Governance

📊

Observability & audit trail

per-call telemetry

Every model call logs bid_id, model+version, prompt_version, tokens, latency, cost to ai_calls + App Insights; raw outputs stored per run → chain bid → prompt_version + raw outputs → reconciled items → estimate → approver.

🧪

Evaluation

golden set + CI gate

Versioned golden set (refrigeration + HVAC) scores recall, status accuracy, size-match; evals run in CI and gate every prompt/model change; the 0.6 reconciler threshold is tuned against it.

🔁

Feedback loop

edit capture

Estimator corrections captured in item_edits, reviewed on a cadence, folded into the golden set to improve prompts.

🔒

Compliance & data governance

managed DPA endpoints

Models via managed, zero-retention endpoints under DPAs (Gemini region-pinned, no training; Claude zero-retention). Prompt/response content redacted before external log sinks; GDPR/CCPA mapped. Models never emit prices.
