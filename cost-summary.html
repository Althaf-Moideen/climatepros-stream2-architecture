What makes up the cost

🤖 AI Model Cost — ~$142/mo

  • Zeus (Claude Sonnet 4.6) — runs on the client's own machine; no cloud charge to TechJays
  • Gemini 3.5 Flash — Google's cloud verifier; very low cost (~$0.01 per bid)
  • Claude Fable 5 — Anthropic's cloud verifier; highest quality, higher token usage (~$0.70 per bid). Can be gated by bid value to reduce cost.

☁️ Cloud Infrastructure — ~$800/mo

  • Azure Container Apps ~$200/mo — runs the API, workers, and background jobs
  • PostgreSQL 16 ~$250/mo — stores all bids, results, and audit logs (HA + daily backups)
  • Knowledge Graph DB ~$150/mo — learns from past corrections over time
  • Redis Cache ~$80/mo — tracks live job status, avoids re-running models
  • Azure Service Bus ~$25/mo — queues bid jobs reliably, retries on failure
  • Blob Storage (incremental) ~$20/mo — extra storage for drawings and results only
  • Monitor + App Insights ~$30/mo — logs, alerts, and performance tracking
  • Networking / misc ~$45/mo — data transfer, private endpoints, DNS

✅ Already paid for by the client — zero extra cost to ClimatePros

Front Door + WAF App Gateway VNet + Private Endpoints Blob Storage FJX Auth Azure AD / Entra ID
These are existing FieldJetX components. ClimatePros reuses them at no extra charge — the $800/mo figure only covers brand-new infrastructure built specifically for this project.

What if we use a different AI model for verification?

💡 Context — how models are used

  • Zeus (Claude Sonnet 4.6) — runs on the client's own machine (on-prem). No cloud charge regardless of which verifier model we pick.
  • Gemini 3.5 Flash — stays the same at ~$0.01/bid in all scenarios. Very low cost.
  • Cloud Verifier (currently Fable 5) — this is the only model we can swap to reduce cost. It accounts for ~98% of all AI spend.

✅ Recommended — Sonnet 4.6

  • Saves ~$116/month in AI costs vs Fable 5
  • Same model family already used by Zeus — proven for this use case
  • Reduces total cost from ~$4.70 to ~$4.13 per bid
  • Can upgrade to Fable 5 / Opus for high-value bids only
  • Good starting point for POC — switch up if accuracy needs improving

⚠️ Trade-off to consider

  • Fable 5 and Opus give better accuracy on complex or unclear drawings
  • Haiku is cheapest but may miss items on dense refrigeration schedules
  • The real question is: how many bids need premium verification?
  • Hybrid approach — Sonnet by default, Fable 5 for bids above $X value — gives best cost/quality balance
Ways to reduce cost if needed

🎛️ Cost control options

  • Skip the Fable 5 verifier for low-value bids — this is the single biggest lever; Fable 5 accounts for ~98% of AI model spend
  • Cache repeated submissions — if the same drawing is re-submitted unchanged, reuse the previous result without re-running any model
  • Reduce image quality for simpler pages — only process dense schedule regions at full resolution
  • Scale down infrastructure floors — right-size the auto-scale minimums based on actual usage patterns
  • Reserved capacity discounts — commit to Azure reserved instances for steady infra to get 20–40% off
