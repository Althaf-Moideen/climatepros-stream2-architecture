Block Reference — What each block does
Clients
FieldJetX Web App The screen the estimator uses to upload drawings and review extracted results
Manager / Admin Manager adds feedback on bids · Admin manages users and rate cards
Existing · FieldJetX Platform
Front Door + WAF The main gate — blocks bad traffic and handles secure connections (TLS)
App Gateway The traffic director — routes requests to the right service
FJX Auth · Session Tokens The security guard — checks who you are and what you're allowed to do on every request
Blob Storage The file cabinet — stores all uploaded drawings, generated quotes and images
VNet + Private Endpoints The private tunnel — keeps all data off the public internet
Key Vault The safe — stores all passwords, API keys and secrets; nothing hardcoded in code
New Infrastructure · This Project
FastAPI + WebSocket API The brain — receives all requests, coordinates processing, sends back results via WebSocket or HTTP polling
Azure Service Bus The queue — holds analyse jobs and delivers them to workers reliably; retries on failure, dead-letters after 3 attempts
KEDA Workers · Celery 5.6 The workforce — scales up automatically, processes all pages in parallel, calls AI models, merges results and runs pricing
Azure SQL Database The main database — stores bids, extracted items, pricing, audit trail and every AI call log · same SQL Server technology as FieldJetX
Azure Cache for Redis The short-term memory — tracks live job progress and caches repeated results so the same drawing is never re-processed
Knowledge Graph DB The learning system — remembers past corrections to make future extractions smarter over time via dynamic few-shot grounding

Observability & Tracing

Why this stack — benefits of newly introduced components
New · Database
Azure SQL Database
  • Same SQL Server engine as FieldJetX — one DB technology for the client to operate and support
  • Client DBA team already knows SQL Server; no extra skill or tooling needed
  • Data lives on Azure alongside the rest of the stack — no cross-cloud hops
  • HA + PITR built in — system stays up if a server fails and can be restored to any point in time
  • Can share or sit alongside the same Azure SQL MI as FieldJetX when the POC moves to production
New · Cache
Azure Cache for Redis
  • Fully managed on Azure — no Redis infrastructure to provision, patch or monitor
  • Tracks live job state so the UI can show real-time progress without polling the database
  • Cache-hit on repeat drawings = zero AI API calls = zero token cost for that request
  • Scales automatically with demand; SLA-backed availability
New · Queue + Workers
Azure Service Bus + KEDA Workers
  • Worker count scales to zero when idle — no compute cost between jobs
  • All pages of a drawing processed in parallel (fan-out), not one by one — faster turnaround
  • Dead-letter queue captures failed jobs for investigation; nothing is silently lost
  • Azure-native queue — no extra broker to manage outside the existing Azure environment
Cloud AI Verifier
Gemini 2.5 Flash
  • Independent second opinion on Zeus extraction — catches errors a single model would miss
  • Runs in parallel with Zeus via asyncio.gather() — adds accuracy with no extra latency
  • Single frontier model keeps architecture simple and cost predictable
  • Google Vertex AI: region-pinned, DPA-covered, no prompt logging — meets data privacy requirements
  • Can be swapped or skipped per job type as cost and performance data accumulates
New · Learning
Knowledge Graph DB
  • Remembers every estimator correction — system gets smarter with each bid reviewed
  • SIMILAR_TO edges link past bid items to new drawings for dynamic few-shot grounding
  • Weekly promo: corrections → golden set → new prompt_version stamped on every AI call
  • Accuracy improves continuously without retraining or manual prompt engineering
Private Model Infra
Zeus — On-prem Extractor
  • Runs on the client's own Mac Studio "Tundra" — drawing data never leaves the client's machine
  • Grounded in prior FieldJetX bid history — output is contextualised to ClimatePros jobs, not generic
  • Constrained JSON schema output — results are always structured and machine-readable, never free-form
  • Models never emit prices — all pricing is pure deterministic code, fully auditable
